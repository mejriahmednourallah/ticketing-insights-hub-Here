# Ticketing Insights Hub: Top-Level Overview

## 1) What This Project Does

Ticketing Insights Hub is a support analytics platform that:

1. Pulls projects and issues from Redmine.
2. Stores and normalizes them in Supabase.
3. Exposes analytics-ready views for the frontend dashboard.
4. Provides similarity analysis between tickets.
5. Adds an AI chat assistant that answers questions using dashboard context.

The product is designed to run either locally (Supabase + Docker web) or with cloud Supabase.

## 3) End-to-End Data Flow

1. Redmine Ingest function fetches all projects from Redmine.
2. It upserts projects into redmine_projects.
3. It processes a project batch and fetches project issues.
4. It maps custom fields into normalized columns (team, technology, type, etc.).
5. It upserts issues into redmine_issues.
6. It updates sync_state cursor and writes execution metrics into sync_runs.
7. Frontend reads from redmine_ticket_view and renders KPI/charts/tables.
8. Similarity page computes similarity scores client-side from loaded tickets.
9. AI chat sends question + ticket summary context to the chat function.
10. Chat function streams model output back to frontend.

## 4) Resumable Ingestion Strategy

To avoid edge runtime CPU cancellation on large datasets, ingestion is resumable:

1. It stores last_offset in sync_state.
2. Each run processes only REDMINE_PROJECT_BATCH_SIZE projects.
3. A run can end as partial with nextOffset for continuation.
4. When the batch cycle reaches the end, it resets to offset 0 and marks cycleCompleted.

Reference implementation:
[supabase/functions/redmine-ingest/index.ts](supabase/functions/redmine-ingest/index.ts)

Key envs:

1. REDMINE_PAGE_SIZE
2. REDMINE_PROJECT_BATCH_SIZE
3. REDMINE_FIELD_* aliases (custom field mapping)

## 8) Why You May See Missing Projects or Empty Dimensions

Common causes:

1. Redmine permission 403 on specific projects.
2. Projects with no issues in Redmine.
3. Ingestion cycle not yet completed (cursor mid-cycle).
4. Custom field alias mismatch (for example, type mapped to wrong Redmine field name).

Recent known mapping note:

Type often comes from Nature or Type d'intervention rather than literal Type.

## 9) Typical Local Execution Flow

1. Configure .env.
2. Run local e2e script.
3. Wait until ingestion cycle reaches success with cursor reset.
4. Open web dashboard and refresh cache.
5. Validate KPIs, project counts, and distribution charts.

## 10) Quick Commands

1. Full local start:
`npm run local:e2e:ps`

2. Force no-cache web build during start:
`powershell -ExecutionPolicy Bypass -File scripts/run-local-e2e.ps1 -NoCache`

3. Skip ingest (infra + web only):
`powershell -ExecutionPolicy Bypass -File scripts/run-local-e2e.ps1 -SkipIngest`

4. Include cloud function deployment stage:
`powershell -ExecutionPolicy Bypass -File scripts/run-local-e2e.ps1 -DeployFunctions`
